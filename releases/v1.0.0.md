xk6-subcommand-httpbin `v1.0.0` is here 🎉!

## Initial Production Release

This is the first production-ready release of xk6-subcommand-httpbin, a k6 subcommand extension that provides a local httpbin server for HTTP testing and load test development.

### Features

- **Local httpbin Server**: Run a fully functional httpbin server directly from k6
- **Simple CLI**: Start with `k6 x httpbin` - no external dependencies required
- **Configurable Binding**: Use `--bind` flag to specify custom host:port (default: localhost:5454)
- **Graceful Shutdown**: Clean server shutdown with Ctrl-C
- **Complete httpbin API**: All standard endpoints including GET/POST/PUT/PATCH/DELETE, status codes, headers, authentication, cookies, redirects, compression, caching, and more
- **Embedded Help**: Access full endpoint documentation via the CLI

### Use Cases

- **Zero-setup Testing**: Test HTTP logic without external services or Docker
- **Fast Local Development**: Eliminate network latency for predictable responses  
- **Offline Development**: Work on k6 scripts without internet connectivity
- **Debugging**: Validate k6 scripts with consistent, deterministic responses
- **Learning**: Example implementation of k6 subcommand extensions

### Installation

Build a custom k6 binary using [xk6](https://github.com/grafana/xk6):

```bash
xk6 build --with github.com/grafana/xk6-subcommand-httpbin@v1.0.0
```

Or download pre-built binaries from the [releases page](https://github.com/grafana/xk6-subcommand-httpbin/releases/tag/v1.0.0).

### Example Usage

**Terminal 1 - Start the server:**
```bash
./k6 x httpbin
```

**Terminal 2 - Run a test:**
```javascript
import http from 'k6/http';
import { check } from 'k6';

export default function () {
    let response = http.get('http://localhost:5454/get');
    check(response, {
        'status is 200': (r) => r.status === 200,
    });
}
```
